Uses a standard approve() + depositBond() two-step. depositBondWithPermit() (gasless approval) is supported on-chain but needs an EIP-2612 signature flow this minimal page doesn't implement yet — see src/settlement-router/client.ts's test suite for a worked example if you're wiring a fuller UI.
Nothing on this page is sent anywhere except the API base URL above and, for agent actions, your connected wallet's own RPC calls. The orchestrator secret you type in the Orchestrator tab is sent only to the API base URL you configure here — treat this page like any other admin console and only point it at an API base you control.
